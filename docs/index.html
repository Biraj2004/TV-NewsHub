3. Stream Architecture & Fallback Edge Cases

How TV-NewsHub fetches streams, handles network failures, and manages TV remote edge cases

Multi-Provider Streaming Engine

Implemented in src/screens/PlayerScreen.tsx

  • Tier 1 (streamUrl): Direct HLS .m3u8 playback via HTML5 video container.
  • Tier 2 (embedUrl): Official share widgets rendered in isolated WebView containers.
  • Tier 3 (youtubeChannelId): Fallback Desktop YouTube Live Resolver.

Live EPG & Program Title Hook

Implemented in src/hooks/useLiveChannelResolver.ts

  • Extracts real-time broadcast headlines (e.g. "Fataak News", "Aaj Banglay").
  • Passes programTitle dynamically to top TV overlay.
  • Auto-updates live status badges across the channel grid.

Smart TV Remote Navigation

Implemented via useTVEventHandler

  • LEFT / RIGHT: Channel surfing zapping (instant previous/next channel).
  • OK / CENTER: Shows/hides translucent TV overlay banner.
  • BACK: Smooth hardware back handler returns to TV grid.

Edge Case Handling & Resilience

CDN Stream Disconnection / 404 Expiration

If a primary HLS stream drops or encounters a CDN token rotation error, the player automatically falls back to Tier 2 (Web Embed) or Tier 3 (YouTube Live Resolver) without crashing.

Ended Broadcasts / Offline Channels

When a live stream ends or reports an offline state, an error overlay appears with a 5-second countdown timer, automatically returning the user to their last focused tile on the home grid.

Cookie Consent Auto-Dismissal

ConsentSafeYouTubePlayer sets Layer 1 consent cookies (CONSENT=YES+) and injects Layer 2 DOM scripts to auto-dismiss consent prompts seamlessly.

Smart TV D-Pad Focus Retention

Includes an invisible FocusGrabber pressable layer above the video player canvas to ensure Smart TV D-Pad focus is never lost to web components.

4. Security Architecture & Privacy Policy

Data protection principles, sandbox isolation, and zero-tracking commitments

Privacy Policy

Zero Personal Data Collection: TV-NewsHub does not track, collect, store, or transmit any user identity, analytics, or personal behavioral data. No user accounts or registrations are required.

Local Storage Only: The application uses AsyncStorage strictly on your local device to persist your last-watched channel and timestamp for 10-minute auto-resume playback.

Security Architecture

Strict HTTPS & Transport Security: All network requests, HLS streams, and API interactions strictly enforce Transport Layer Security (TLS 1.2+ / HTTPS).

Sandboxed Web Execution: Embedded web share widgets run within isolated WebViews with restricted DOM access to prevent cross-origin script injection or unauthorized access.
